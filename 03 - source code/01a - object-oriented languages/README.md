At first, I hesitated to sort [(GNU) Smalltalk](https://github.com/practicalcomputerscience/MicrobenchmarkGPHLlanguages/tree/main/03%20-%20source%20code/01a%20-%20object-oriented%20languages/Smalltalk#gnu-smalltalk) into the same category as Eiffel, because, and at least with [GNU Smalltalk](https://www.gnu.org/software/smalltalk/),
it's possible to print "Hello, World!" on the console without any class definition, something which isn't possible with the later (Liberty) Eiffel: [Idea of Eiffel](https://github.com/practicalcomputerscience/MicrobenchmarkGPHLlanguages/tree/main/03%20-%20source%20code/01a%20-%20object-oriented%20languages/Eiffel#idea-of-eiffel)

However, Smalltalk-72, right at start, was designed around the central idea of object-orientation:

> Though it has noble ancestors indeed, Smalltalk’s contribution is a new design paradigm - which I called object-oriented - for attacking large problems of the professional programmer, and making small ones possible for the novice user.

from: [The Early History of Smalltalk](https://www.researchgate.net/publication/221501755_The_Early_History_of_Smalltalk) by Alan Kay, 1993
